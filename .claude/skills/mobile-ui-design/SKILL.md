# 모바일 UI 디자인 스펙

## 언제 사용하는가
모바일 우선 웹앱의 화면 스펙과 디자인 시스템을 문서로 만들어야 할 때. 실제 CSS 코드가 아닌 **의사결정 문서**를 산출한다.

## 산출 문서 구조

### 1. 요약 (1줄)
앱의 성격을 한 문장으로. 이 문장이 이후 모든 색·톤·문구 선택의 나침반.

### 2. 페르소나 (3~5줄)
- 주 사용자: 누구인가
- 사용 상황: 언제/어디서 열게 되는가
- 감정: 앱을 열 때의 심리 상태

이 세 가지가 UI 톤을 결정한다. 예: "불쾌한 상황에서 열게 되므로 앱 자체는 침착하고 감정을 자극하지 않는 톤."

### 3. 정보구조 (IA)
화면 목록과 이동 흐름을 다이어그램으로.
```
[홈=신고] ⟷ [통계] ⟷ [단지관리]
    │
    └→ [강도 선택 오버레이] → [신고 완료 토스트]
```

### 4. 화면별 스펙
각 화면마다:
- **목적**: 이 화면에서 사용자가 이루려는 것 한 줄
- **레이아웃**: 텍스트 wireframe
  ```
  ┌─────────────────────┐
  │ [☰]  냄새 리포트  [⚙] │  헤더 56px
  ├─────────────────────┤
  │  단지: [샘플아파트▼]  │  선택 바
  │                     │
  │      🌊             │  중앙 시각 요소
  │   지금 냄새가        │
  │    나나요?           │
  │  ┌──────────────┐   │
  │  │ 지금 냄새 남 │   │  주 CTA 200x64
  │  └──────────────┘   │
  ```
- **주요 상태**: normal / empty / loading / error
- **인터랙션**: 탭 시 무엇이 일어나는가

### 5. 디자인 토큰
```yaml
color:
  brand:          "#XXX"   # 주요 액션
  ink:            "#XXX"   # 본문
  ink-muted:      "#XXX"
  bg:             "#XXX"
  surface:        "#XXX"   # 카드 배경
  border:         "#XXX"
  intensity-1:    "#XXX"   # 약함
  intensity-2:    "#XXX"   # 보통
  intensity-3:    "#XXX"   # 강함
  success:        "#XXX"
  danger:         "#XXX"
font:
  stack: "system-ui, -apple-system, 'Segoe UI', Roboto, 'Apple SD Gothic Neo', sans-serif"
  scale: 12, 14, 16, 20, 24, 32
space: 4, 8, 12, 16, 24, 32, 48
radius: 8, 12, 24
shadow:
  sm: "0 1px 2px rgba(0,0,0,.06)"
  md: "0 4px 12px rgba(0,0,0,.08)"
```

라이트/다크 각각 명시. HEX로.

### 6. 핵심 컴포넌트
문서화가 값진 것만:
- 주 CTA(신고 버튼) — 크기, 강도 선택 상호작용
- 히트맵 카드 — 데이터 없을 때 UI
- 단지 선택 드롭다운 — 검색 + 추가 옵션

### 7. 접근성 규칙
- 터치 영역 44x44px 이상
- 텍스트 대비비 4.5:1 이상 (본문)
- 색만으로 정보를 전달하지 않는다 (아이콘/라벨 병기)
- 폼 라벨과 인풋 연결

### 8. 문구 가이드
톤과 예시:
- 헤더: 담백. "냄새 리포트"
- CTA: 즉각적. "지금 냄새 남"
- 빈 상태: 격려. "아직 신고가 없어요. 첫 신고자가 되어보세요."
- 에러: 상황 설명 + 다음 행동. "저장이 안 됐어요. 잠시 후 다시 시도해주세요."

## 원칙

| 원칙 | 이유 |
|------|------|
| Why-first | 각 결정에 "왜"를 붙여야 이후 편집자가 무너뜨리지 않는다 |
| 모바일 우선 | 스마트폰 화면부터 그리고 큰 화면은 max-width로 확장만 |
| 원터치 지향 | 주요 액션은 1탭 완료. 강도 선택 같은 2단계도 즉시 노출된 라디오 UI |
| 감정 톤 유의 | 신고 앱은 자극적으로 만들면 반복 사용을 유도해 데이터 왜곡 위험 |
| 색맹 대응 | 특히 "강도" 표현은 색+숫자+라벨 3중으로 |

## 자주 하는 실수
- 데스크톱 우선 설계 후 축소 → 모바일에서 어색
- 브랜드 컬러를 강도 색상에 사용 → 데이터 표현 혼란
- 빈 상태 스킵 → 초기 사용자가 무엇을 해야 할지 모름
- 로그인 UI를 은근히 남겨둠 → 익명 앱에서는 흔적도 지운다
