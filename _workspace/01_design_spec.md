# 신안APT 냄새 리포트 — 디자인 스펙 v2

## 1. 요약
Linear 다크 시스템 위에 지도 기반 익명 냄새 리포트 앱.
**Why:** 이 한 줄이 이후 모든 색/톤/문구 선택의 나침반. "지도 기반"은 12개 동을 시각적으로 선택한다는 뜻이고, "Linear 다크"는 브랜드/CTA/포커스에만 라벤더를 아껴 쓰는 소프트웨어 크래프트 톤을 뜻한다.

---

## 2. 페르소나
- **주 사용자:** 효천마을 신안인스빌 1·2단지 30~60대 주민. 앱 설치 없이 URL로 접속.
- **사용 상황:** 창문 열다·산책 중 개울에서 악취 감지 순간. 대부분 한 손 조작.
- **감정 상태:** 불쾌·짜증. 3초 안에 신고 완료를 원함. 자극적 UI 시 회피.
- **부가 시나리오:** "어제 저녁 냄새 났는데 이제 신고" — 30일 이내 소급 신고 지원.
- **최종 목적:** ① 이웃과 "우리만 그런 거 아니구나" 공유, ② 구청 민원 자료로 사용.

**Why:** 목적이 이웃 공유(감정)와 민원 자료(정확도) 두 가지이기 때문에, UI는 담백하되 날짜·건물 정확도를 시각적으로 드러내야 함.

## 3. 정보구조 (IA)

```
        [진입 = 홈/신고 (지도)]
             │
             ├─ 날짜 선택 (오늘~-30일)
             ├─ 동 선택 (지도 탭)
             ├─ 강도 선택 (1/2/3)
             └─ CTA → 신고 완료 토스트
             │
             ▼
      [통계 대시보드]
             ├─ KPI 3칸 (오늘/이번 주/이번 달)
             ├─ 12행 x N일 히트맵
             ├─ 동별/단지별 토글
             └─ 리포트 내보내기 (CSV + 인쇄용 요약)
```

- 하단 탭바 **2개**: `홈 · 통계`.
- **Why 2탭:** 대상 아파트가 고정(12동)이라 단지 관리 화면이 불필요. 뎁스를 얕게 유지해 이탈 감소.

### 4.1 홈 (지도 신고 화면)

**목적:** 지도 위 내 동을 탭하고 강도를 골라 3초 안에 신고.

**주요 상태:**
- `normal`: 지도 로드 완료, 마지막 선택 동 자동 하이라이트.
- `no-building`: 동 미선택 → CTA 탭 시 지도 상단에 인라인 안내 "동을 먼저 골라주세요".
- `no-intensity`: 강도 미선택 → CTA 탭 시 세그먼트 옆 인라인 안내.
- `past-date`: 날짜가 오늘이 아님 → CTA 라벨 "M월 D일 신고하기"에 caption 뱃지 "과거 날짜" 라벤더 tint pill.
- `submitting`: CTA "보내는 중…" + 스피너, disabled.
- `success`: 하단 토스트 "신고가 접수됐어요. 감사합니다." 3초 후 자동 리셋(강도만).
- `error`: 토스트 "지금 전송이 안 됐어요. 잠시 후 다시 시도해주세요." + CTA 재활성.
- `empty`: 최근 신고 자리에 "이 지도의 첫 신고자가 되어보세요".
- `map-loading`: 지도 컨테이너에 surface-2 스켈레톤 + shimmer.

**인터랙션:**
- **동 원형 탭:** 즉시 선택 → 라벤더 fill @20% + hairline-strong stroke, localStorage에 `lastBuildingId` 저장.
- **뷰 토글(오늘/이번 주):** 각 동 뱃지 숫자 즉시 갱신 (동별 신고 카운트).
- **날짜 선택기 탭:** native `<input type="date">` 열림 (min=오늘-30, max=오늘). 오늘 아니면 헤더 라벨이 "📅 7월 3일" + 라벤더 tint.
- **강도 세그먼트 탭:** 즉시 선택 (하이라이트).
- **CTA 탭:** 동·강도 모두 있어야 활성 시각. 미충족 시 shake + 인라인 안내.
- **재방문:** localStorage `lastBuildingId` 있으면 그 동 자동 선택, 날짜는 항상 오늘로 리셋.
- **Why 자동 저장:** 대부분 사용자가 "내 동"만 반복 신고 → 매 방문 재선택 마찰 제거.
- **Why 날짜는 리셋:** 소급 신고는 예외적 시나리오 → 기본은 항상 오늘이라야 사고 방지.

## 6. 컴포넌트 스펙

### 6.1 지도 오버레이 (지도 이미지 + SVG 히트 영역)

**컨테이너:**
- surface-1 배경, rounded-xl 16px, hairline 1px 보더.
- `aspect-ratio: 2278 / 937` 유지 (max-width 100%).
- 내부: `<img src="assets/map.jpg">` + 그 위 절대 위치 `<svg viewBox="0 0 2278 937" preserveAspectRatio="xMidYMid meet">`.

**뷰 토글 (컨테이너 상단):**
- pill 세그먼트 2분할 `[오늘 | 이번 주]`, surface-2 배경, 선택된 것만 ink.
- 우측에 총 건수 status-badge (pill, caption).

**동 원형 히트 영역 (`<circle>` × 12):**
- 좌표: `_workspace/00_user_feedback.md`의 `BUILDINGS` 초안 그대로.
- 기본:
  - `stroke=hairline` 2px, `fill=transparent`
  - 중앙 `<text>` 동 번호 (13px 500, ink, tracking +0.4)
  - 상단 12시 방향 (cy - r - 12) 신고 건수 status-badge
- Hover/포커스: `stroke=hairline-strong` 2px
- 선택됨: `fill=primary` @20% (rgba), `stroke=primary` 2px, 텍스트 색 유지
- 접근성: 각 `<g>`에 `role="button"`, `aria-label="103동, 오늘 4건 신고"`, `tabindex="0"`.
- **Why SVG:** 반응형 자동 스케일. HTML `<map>`은 반응형 좌표 지원 부족.

### 6.2 날짜 선택기 (헤더)

**트리거 (헤더 우측):**
- pill status-badge 스타일: surface-2 배경, caption 텍스트, 아이콘 📅 + 라벨.
- 오늘: 라벨 "오늘", ink-muted.
- 과거일: 라벨 "M월 D일", primary tint (라벤더 배경 @15%, ink 텍스트).

**동작:**
- 탭 시 native `<input type="date">` 열림 (`display: none`, 프로그래매틱 `click()`).
- `min = today - 30days` (YYYY-MM-DD), `max = today`.
- 선택 후 헤더 라벨 즉시 갱신 + CTA 라벨 갱신.
- 30일 초과 접근 (URL 파라미터 등) 시 → 인라인 토스트 "최대 30일 전까지 신고할 수 있어요" + 오늘로 리셋.
- **Why native picker:** 라이브러리 무의존, iOS/Android 모두 무료로 접근성 확보.

### 6.3 강도 세그먼트

**구조:** 3분할 세그먼트 컨트롤 (grid 1fr 1fr 1fr).
- 각 셀 44px 높이, surface-1 배경, hairline 보더 (첫/마지막 셀만 좌/우 라운드 md 8px, 가운데는 각).
- 미선택: `bg=surface-1`, 텍스트 ink-muted, 상단 숫자 subhead 20/500, 하단 라벨 caption.
- 선택: `bg=surface-2`, `border=hairline-strong`, 숫자에 강도-N 색상, 라벨 ink.
- **Why 강도 색은 텍스트에만:** 배경을 강도 색으로 채우면 CTA와 시각적 우선순위 충돌. Linear는 필-크로마 최소화.

### 6.4 CTA "지금 냄새 남"

- 100% x 48px, rounded-md 8px.
- `bg=primary (#5e6ad2)`, `text=#ffffff`, button typography 14/500.
- 활성: 동+강도 모두 선택됨.
- 비활성 시각: opacity 0.5, cursor not-allowed. **탭 자체는 허용 → 미충족 필드로 shake + 인라인 안내.**
- 과거일 라벨: `M월 D일 신고하기` + 우측에 라벤더 tint pill "과거 날짜" (caption).
- pressed: scale 0.98, 120ms.
- focus: 2px `primary-focus` outline (offset 2px).
- **Why pill 아닌 md 라운드:** Linear 원칙 — pill CTA 금지. 8px 라운드 유지.

### 6.5 KPI 카드

- 3분할 grid, 각 셀 surface-1 배경 + hairline 보더 + rounded-lg 12px + 24px 패딩.
- 상단 숫자 headline 28/600, tabular-nums.
- 하단 라벨 caption 12/400 ink-muted.
- 데이터 0일 때: 숫자만 표시, 라벨 유지 (empty state와 시각 통일).
- **Why 3칸:** 오늘/이번 주/이번 달 — 사용자의 시간 감각 3단계와 매핑.

### 6.6 히트맵

**그리드:**
- `display: grid`, 첫 열은 동 이름 라벨 (60px 고정), 이후 N열 (기간에 따라 7/30/365).
- 각 셀 12x12 (모바일 최소) ~ 20x20 (넓은 화면). `min-width: 12px`.
- 가로 스크롤 허용 (30일/올해 뷰).
- 셀 색상: 강도 평균 → cell-0/1/2/3 매핑.
- 접근성: 각 셀 `<button aria-label="103동, 7월 3일, 4건, 평균 강도 2">`.

**범례 (필수):**
- 하단에 `없음 · 약함 · 보통 · 심함` 캡션 + 색 샘플 병기.

**Why cell 색 매핑 규칙:** Linear의 서페이스 라더를 강도 강도에 재활용 → 새 브랜드 색 도입 없이 위계 표현.

### 6.7 리포트 내보내기 버튼

- secondary 스타일: surface-1 배경, hairline 보더, ink 텍스트, rounded-md.
- 100% x 44px, 좌측 아이콘 📥.
- 탭 시 바텀시트 (surface-3 배경, rounded-t-xl):
  - 기간 선택 (기본은 현재 활성 기간).
  - 형식 선택: `CSV 다운로드` / `인쇄용 요약`.
  - primary CTA "다운로드".
- CSV: `Blob` + `URL.createObjectURL`로 클라이언트 다운로드.
- 인쇄: 새 창에 인쇄 전용 HTML 렌더 후 `window.print()`.
- **Why 바텀시트:** 옵션 여러 개(기간/형식)를 원자적으로 선택. 인라인 폼은 세로 공간 낭비.

### 6.8 하단 탭바

- height 64px, canvas 배경 + hairline 상단선.
- 2칸 grid, 각 칸에 아이콘 + 라벨.
- 활성: 아이콘 primary, 라벨 ink.
- 비활성: 아이콘 ink-subtle, 라벨 ink-muted.
- 터치 영역 각 44px 이상.

## 7. 접근성

- **터치 영역 44x44px 이상:** 동 원(r=70~100 → 원본 픽셀 기준. 화면 스케일에서도 hitbox 44 이상), 강도 세그먼트, CTA, 탭바, 지도 뱃지 모두.
- **대비비 WCAG AA:**
  - 다크: ink #f7f8f8 on canvas #010102 = 19:1. ink-muted on canvas = 12:1.
  - 라이트: ink #0d0e10 on canvas #fafafa = 18:1.
  - 강도 색상은 라이트/다크 모두 4.5:1 이상 검증.
- **색 단독 금지:** 강도는 색 + 숫자 + 라벨 3중. 히트맵 범례 필수.
- **포커스 링:** 2px `primary-focus` outline, offset 2px. 브라우저 기본 outline 제거 금지.
- **ARIA:**
  - 지도 SVG: `role="img" aria-label="효천마을 신안인스빌 전체 지도"`.
  - 동 원: `role="button"` + 상태별 aria-label.
  - 세그먼트: `role="radiogroup" aria-label="냄새 강도"`, 각 셀 `role="radio"`.
  - 날짜 pill: `aria-label="신고 날짜 선택, 현재 오늘"`.
  - 히트맵 셀: `role="button"` + 상세 aria-label.
  - 토스트: `role="status" aria-live="polite"`.
- **키보드:** Tab 순서 지도→강도→CTA. 지도 내에서는 좌→우, 위→아래 순.
- **모션 감소:** `prefers-reduced-motion` 시 shake/scale 무효, opacity 페이드만.
- **폰트 크기:** rem 기반. 200% 확대에도 지도/히트맵 스크롤 가능.
- **Why:** 60대 사용자, 야외 밝기, 색맹 3중 리스크. Linear는 데스크톱 시스템이므로 모바일 접근성은 별도 강화.
