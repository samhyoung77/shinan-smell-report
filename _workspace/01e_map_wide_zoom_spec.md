## 확대/축소 + 이동 (신규 인터랙션)
현재 `.map-wrap` 가로 스크롤 방식을 **transform 기반 줌/팬**으로 교체.
- 구조: `.map-wrap`(고정 높이, overflow:hidden, position:relative) > `.map-inner`(transform 대상, bg svg + overlay svg 포함, transform-origin: 0 0).
- 상태: `scale`(1~5), `tx`,`ty`(px). 기본 `scale=1`(전체가 보이도록 fit-width) → **처음엔 개울 포함 전체 조망**.
- 조작:
  - **버튼**: 우하단(또는 우상단) `＋ / －` (선택적 `⟳ 리셋`). 접근성 aria-label.
  - **휠**: 커서 기준 줌.
  - **드래그(팬)**: pointerdown+move로 tx,ty 갱신. 이동 임계(예 6px) 초과 시 '드래그'로 간주.
  - **핀치(터치)**: 두 포인터 거리로 줌(중심 기준).
  - **더블탭/더블클릭**: 해당 지점 확대(선택).
- **선택과 충돌 방지**: pointerup 시 이동량이 임계 미만이면 '탭'→ 그 지점의 `.b` 선택(기존 onSelect). 드래그면 선택 안 함. 기존 키보드 선택(원 focus + Enter/Space) 유지.
- **클램프**: tx,ty를 지도가 뷰포트를 항상 덮도록 제한(빈 여백 방지). scale=1이면 tx=ty=0.
- **선택 동 자동 센터**: 초기 selectedId 있으면 그 동이 보이도록 팬(가능하면 살짝 확대). 기존 scrollLeft 로직 대체.
- 성능: transform만 변경(리플로우 최소). rAF로 처리.

## 배치/치수
- `.map-wrap` 높이: aspect 2039:1025 유지하되 모바일에서 너무 낮지 않게(최소 높이 예 240px, 또는 aspect-ratio + max-height). 데스크톱 480px 컬럼 기준 확인.
