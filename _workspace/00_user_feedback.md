# 사용자 피드백 및 도메인 정보 (2026-07-04)

## 지형/위치
- 사용자가 구글맵 캡쳐 이미지를 제공할 예정.
- 이미지 위에 나타난 아파트 단지들을 **시각적으로 선택**할 수 있어야 함.
- 드롭다운 방식 폐기 → **이미지 맵 방식**으로 단지 선택.

**구현 방향 (frontend-builder 참고):**
- 이미지를 `assets/map.png` (또는 유사)로 저장
- 각 아파트 단지 영역을 다각형(polygon) 좌표로 매핑 → SVG overlay 또는 `<map>`/`<area>` 태그
- 좌표 데이터는 `js/complexes-map.js`에 배열로 정의 (초기 하드코딩, 이후 콘솔에서 추가 가능하도록)
- 단지를 탭하면 하이라이트되고 선택 상태가 됨. 헤더에 "선택된 단지: {name}" 표시.
- 이미지 위에는 신고 건수 뱃지 표시 (오늘/이번 주 토글)

## 냄새 유형
- 유형 구분 없음. "냄새 남" 단일 이벤트 + 강도 1/2/3.
- 기존 스펙 유지.

## 신고 날짜 선택 (신규 추가)
- 기본값은 **오늘 날짜**로 자동 세팅.
- 헤더/신고 카드에 **달력 아이콘** 노출 → 탭 시 date picker 열리고 다른 날짜 선택 가능.
- 사용 시나리오: "어제 저녁에 냄새 났는데 이제 신고" 케이스 지원.

**구현 결정 (frontend-builder 참고):**
- `<input type="date">` native picker 사용 (iOS/Android 모두 지원, 별도 라이브러리 불필요).
- **과거 범위 제한: 최대 30일 전까지만.** 이유:
  1. 익명 신고 특성상 오래된 조작 방지
  2. 통계 관점에서도 최근 데이터가 유용
  3. 오늘 기준 `min` attribute로 강제
- **미래 날짜 불가.** `max=오늘`.
- 선택된 날짜가 오늘이 아니면 신고 버튼에 뱃지 표시 ("○월 ○일 신고 중") — 실수 방지.
- Firestore 스키마: 기존 `clientDate`(YYYY-MM-DD)를 그대로 사용. 다만 `ts`(serverTimestamp)와 별개로 사용자가 지정한 날짜를 담는다.
- **`ts`는 서버 시각 그대로** (실제 기록 시각), **`clientDate`는 사용자 선택 날짜**. 통계는 clientDate 기준.
- 30일 초과 요청 시 클라이언트에서 차단 + 안내 문구.
